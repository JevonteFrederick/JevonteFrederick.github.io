# Contact-Service-Project

I can ensure my code is functional and secure by running unit tests that aim to satisfy its requirements. Junit test files are an effective method to build unit tests. I used it to make unit tests for individual requirements of the program that must pass with expected results before I complete the program.

I interpret user needs as mandatory aspects of the program. I incorporate them as the basic functionality of my program, and my unit tests focus on verification that the user needs are met. It's important to integrate user needs as it's essential to delivering quality software. I don't consider a program complete unless I accounted for all user needs I have to consider.

I approach designing software as a puzzle. I try to find the most efficient solution to the problem. In the case of this project, the puzzle was designing classes that are organized and meet the requirements of the contact service program. Another aspect was understanding how I could build unit tests to test the functionality of each unit of my code. Once I find a solution, I make sure to refactor my code so that it meets all requirements and is easy to trace the functionality of.
